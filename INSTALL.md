# Install Bristol OS

## The only step (for anyone at Bristol)
1. Open the **Claude desktop app**.
2. Paste this message and send it:

> **Install Bristol OS for me. Fetch this file and follow every instruction in it exactly, then walk me through setup step by step in plain English: https://raw.githubusercontent.com/Fusion-Data-Company/bristol-os/main/START-HERE.md**

That's it. Claude does the rest — pulls the pack, sets up your workspace, recognizes who you are, writes your `CLAUDE.md`, builds your live Deal Tracker, speaks a welcome, and runs a real task so you can see it. ~5–10 minutes. No accounts, no keys, no command line.

## What it sets up automatically
- A personalized `CLAUDE.md` (your role, markets, deal criteria, format preference) + persistent **memory** so it remembers you next time.
- The research **data feeds** (already keyed): parcels via Quarry, plus FRED, Census, Tavily, Exa, Firecrawl, and voice — nothing to paste.
- Your live **Deal Tracker** and the document/visual tools (Word, Excel, PDF, slides, infographics).
- It will **ask what tools Bristol already uses** (CoStar, Yardi, your model, CRM) and connect them or set up an export path — so the analysis reflects Bristol's real numbers.

## Requirements
- The **Claude desktop app** (Cowork mode) so it can save files to a folder. A plain browser chat can't save your workspace.
- Nothing else for the core. (Optional power-layer MCP servers need Node/uv/Docker — see [`config/CONFIG-GUIDE.md`](config/CONFIG-GUIDE.md); that's for a technical helper, not required.)

## After install
Open your Bristol OS folder and just talk: *"Who owns this parcel?"*, *"Is this site feasible? Build the packet."*, *"Pull rent comps and a market study."*, *"What's on my plate?"* Every analysis is produced to institutional rigor and fully cited.
